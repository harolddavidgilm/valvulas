# Documento de Especificaciones Técnicas
## Aplicativo Web CMMS para Gestión de Válvulas PSV / PRV
**Versión:** 1.0
**Normativas Aplicables:** ASME Sección I y VIII, API 510, API 576, API 527.

---

## 1. Introducción y Alcance
El aplicativo web tiene como propósito funcionar como un Sistema Computarizado de Gestión de Mantenimiento (CMMS) orientado específicamente a la integridad mecánica y el ciclo de vida de válvulas de alivio de presión (PRV) y válvulas de seguridad (PSV) [1, 2]. El sistema asegurará que los dispositivos se abran a la presión especificada dentro de las tolerancias de los códigos aplicables [3], automatizando rutinas de mantenimiento, cálculos de calibración y análisis de Inspección Basada en Riesgo (RBI) [4].

## 2. Arquitectura y Stack Tecnológico
*   **Frontend:** React o Next.js.
*   **Backend & Base de Datos:** PostgreSQL (vía Supabase) para asegurar integridad relacional [5].
*   **Diseño UX/UI Industrial:** La interfaz debe diseñarse considerando el entorno real del usuario. Se deben contemplar factores como el uso de pantallas táctiles con guantes de protección y condiciones de presión o ruido en planta [6]. El diseño priorizará el principio de Pareto, destacando el 20% de las funcionalidades que los técnicos usarán el 80% del tiempo [7].

## 3. Módulos Principales

### 3.1. Gestión de Activos (Hoja de Vida)
Constituye el núcleo del sistema, basado en una jerarquía de activos lógica (ej. estándar ISO 14224) [8].
*   Registro único de Válvula (Tag, ID) [9].
*   Tipo: PSV (típicamente gas/vapor) o PRV (típicamente líquidos) [10].
*   Parámetros de Diseño: Tamaño de entrada/salida, rating ANSI, fabricante, fluido de servicio [9].
*   Parámetros Operativos: Presión de operación, Presión de Set, CDTP (Presión de Prueba Diferencial en Frío) y MAWP (Máxima Presión de Trabajo Permitida del equipo protegido) [9, 11, 12].

### 3.2. Gestión de Mantenimiento y Órdenes de Trabajo (OT)
*   Creación de planes de mantenimiento preventivo y correctivo [13].
*   Generación automática de OT según frecuencias definidas [13].
*   Control de repuestos y tiempos de intervención [14].

### 3.3. Módulo de Pruebas y Calibraciones
Registro de pruebas realizadas en sitio o en banco de pruebas (*Test Stand*) [14, 15]:
*   **Prueba de Disparo (Pop Test):** Aumento lento de la presión (aire, agua o nitrógeno) hasta que la válvula dispare [16], calculando automáticamente el porcentaje de error respecto al Set Pressure.
*   **Prueba de Fugas en Asiento (Leak Test):** Basado en API 527. La presión se mantiene al 90% de la presión de set (o 5 psi por debajo si el set es < 50 psi) [17-19]. El sistema validará la tasa de fuga ingresada (burbujas por minuto o cc/hr) contra el criterio de aceptación normativo [20].

### 3.4. Módulo RBI (Inspección Basada en Riesgo API 510/576)
Reemplaza las frecuencias estáticas por intervalos dinámicos basados en riesgo [4, 21].
*   **Probabilidad de Falla (POF):** Evaluada (Score 1-5) según historial de fallas, edad, pérdida de metal/corrosión remanente [22, 23].
*   **Consecuencia de Falla (COF):** Evaluada (Score 1-5) según impacto en seguridad, costo económico, y peligrosidad del fluido [23].
*   **Cálculo:** `RIESGO = POF × COF` [21].
*   **Ajuste Dinámico:** El sistema reasigna el intervalo de mantenimiento automáticamente (ej. Crítico: 6-12 meses, Bajo: 36-60 meses) [21].

## 4. Estructura de la Base de Datos (Modelo Relacional PostgreSQL)

*   **`valvulas`:** `id_valvula` (PK), `tag_planta`, `tipo`, `fluido`, `tamano_in_out`, `presion_set`, `cdtp`, `mawp`, `estado` [9].
*   **`intervenciones_historico`:** `id_intervencion`, `valvula_id` (FK), `fecha`, `tipo` (Inspección, Prueba, Mantenimiento), `tecnico`, `observaciones` [13].
*   **`planes_mantenimiento`:** `id_plan`, `valvula_id` (FK), `frecuencia_base`, `proxima_fecha` [13].
*   **`ordenes_trabajo`:** `id_ot`, `valvula_id` (FK), `estado`, `fecha_programada` [13].
*   **`pruebas_calibracion`:** `id_prueba`, `valvula_id` (FK), `tipo_prueba`, `presion_disparo_real`, `porcentaje_error`, `tasa_fugas`, `resultado` (PASA/FALLA) [14].
*   **`rbi_analisis`:** `id_rbi`, `valvula_id` (FK), `score_pof`, `score_cof`, `nivel_riesgo`, `intervalo_recomendado` [4, 21, 23].

## 5. Algoritmos y Reglas de Negocio (Backend)

El sistema validará automáticamente los resultados de calibración (Pop Test) dependiendo de la norma de construcción del equipo protegido:

### 5.1. Tolerancias ASME Sección VIII (Recipientes a Presión)
*   Para Presión de Set $\le$ 70 psi: La tolerancia permitida es **$\pm$ 2 psi** [24, 25].
*   Para Presión de Set $>$ 70 psi: La tolerancia permitida es **$\pm$ 3%** de la presión de set [24, 25].

### 5.2. Tolerancias ASME Sección I (Calderas de Potencia)
*   Para Presión de Set $\le$ 70 psi: **$\pm$ 2 psi** [24, 25].
*   Para Presión de Set de 71 a 300 psi: **$\pm$ 3%** [24, 25].
*   Para Presión de Set de 301 a 1000 psi: **$\pm$ 10 psi** [24, 25].
*   Para Presión de Set $>$ 1000 psi: **$\pm$ 1%** [24, 25].

### 5.3. Validación de Blowdown (Diferencial de Cierre)
*   **ASME I:** El sistema exigirá un diferencial de cierre mínimo de 4 psi (para sets $\le$ 67 psi), y escalonadamente hasta 4% para presiones $>$ 375 psi [24, 25].
*   **ASME VIII:** Aunque la norma no tiene un requisito estricto en servicio [24], el sistema validará como advertencia un estándar de la industria fijado entre el **7% y 10%** de la presión de set [25, 26].

## 6. Automatizaciones, Alertas y KPIs (Dashboards)

*   **Disparadores (Triggers):** Recálculo automático de la matriz de riesgo (POF x COF) si se registra un mantenimiento, una falla, o cambia una condición de proceso [27].
*   **Alertas:** Generación de alertas por vencimiento de pruebas, riesgo alto/crítico detectado, o válvulas que repetidamente quedan fuera de rango ("Malos actores") [27, 28].
*   **KPIs del Dashboard:** 
    *   MTBF (Tiempo Medio Entre Fallas) [29].
    *   Porcentaje de cumplimiento del plan de mantenimiento [29].
    *   Distribución visual de válvulas críticas en matriz de riesgo 5x5 [21, 30].
